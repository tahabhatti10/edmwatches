# Rolex Day-Date 40 — Scroll-Driven Landing Page

A luxury watch landing page with canvas-based scroll-scrubbing hero,
powered by Higgsfield AI-generated video frames.

---

## Stack

- **Next.js 15** (App Router)
- **React 19**
- **Framer Motion** — scroll-triggered animations
- **TypeScript**
- Fonts: Playfair Display (display) + Inter (body)

---

## Step-by-Step Setup

### 1. Install dependencies

```bash
npm install
```

### 2. Generate the hero video in Higgsfield

Use the prompts in the original brief to generate:
1. **Asset 1** — base product image (Nano Banana Pro)
2. **Asset 2** — exploded-view reference (Nano Banana Pro, using Asset 1 as reference)
3. **Asset 3** — hero video (Seedance 2.0, 10 s, 16:9, using both images as references)

Download the video as **`hero.mp4`** and place it in this project's root.

### 3. Extract frames

```bash
bash extract-frames.sh
```

This outputs `public/frames/frame_0001.jpg … frame_XXXX.jpg` and prints
the total frame count.

### 4. Update FRAME_COUNT

Open `app/components/ScrollHero.tsx` and set:

```ts
const FRAME_COUNT = 240   // ← replace with your actual count
```

### 5. Run dev server

```bash
npm run dev
```

Open http://localhost:3000

### Verify

| Check | Expected |
|-------|----------|
| http://localhost:3000 | Loads — frame 0 fills viewport |
| http://localhost:3000/frames/frame_0001.jpg | Returns 200 |
| Scroll through hero | Frames scrub assembled → exploded |
| Scroll past hero | Features / Specs / CTA animate in |
| Mobile 375px | Grid stacks to single column |

## Notes

- The scroll hero uses `requestAnimationFrame` + `getBoundingClientRect()`.
  No scroll event listeners.
- Frames are preloaded eagerly; only decoded frames are drawn.
- Canvas uses `devicePixelRatio` scaling — sharp on Retina displays.
- All copy is real luxury watch copy — zero lorem ipsum.
